### 操作步骤

#### 步骤 1.1: 输入密码

在 **"密码"** 输入框中输入您的密码：

**弱密码示例** (不推荐):
```
12345678      ◀─ 纯数字，强度：弱（红色 33%）
password      ◀─ 纯字母，强度：弱（红色 33%）
```

**中等密码示例** (可接受):
```
MyPass123     ◀─ 字母+数字，强度：中等（橙色 66%）
Hello2024     ◀─ 大小写+数字，强度：中等（橙色 66%）
```

**强密码示例** (推荐):
```
MySecure@Pass2024    ◀─ 完整字符集，强度：强（绿色 100%）
BlockChain#2024      ◀─ 大小写+数字+特殊，强度：强（绿色 100%）
```

#### 步骤 1.2: 观察密码强度

输入密码后，会立即看到强度指示器：

**弱 (Weak)**
```
密码强度                    弱
████░░░░░░░░░░░░░░░░░░      ◀─ 红色，33% 宽度
```

**中等 (Medium)**
```
密码强度                    中等
████████████░░░░░░░░░░      ◀─ 橙色，66% 宽度
```

**强 (Strong)**
```
密码强度                    强
████████████████████████    ◀─ 绿色，100% 宽度
```

#### 步骤 1.3: 确认密码

在 **"确认密码"** 输入框中再次输入相同密码：

**匹配状态**:
- ✓ 密码匹配 (绿色) ◀─ 两次输入一致
- ✗ 密码不匹配 (红色) ◀─ 两次输入不一致

#### 步骤 1.4: 点击继续

当满足以下条件时，**"继续"** 按钮变为可点击：
- ✅ 密码至少 8 位
- ✅ 确认密码一致

点击按钮后：
1. 系统保存密码
2. 自动生成 12 个助记词
3. 跳转到备份助记词页面

## 密码强度详解

### 如何计算强度？

```javascript
弱 (Weak):
- 长度 < 8 位
示例: 1234567

中等 (Medium):
- 长度 >= 8 位
- 包含字母和数字
示例: Pass1234, hello2024

强 (Strong):
- 长度 >= 12 位
- 包含大写字母
- 包含小写字母
- 包含数字
- 包含特殊字符 (!@#$%^&* 等)
示例: MySecure@Pass2024
```

### 密码强度对比表

| 密码示例 | 长度 | 字符类型 | 强度 | 颜色 | 宽度 |

| password | 8 | 小写字母 | 弱 | 🔴 红色 | 33% |
| Pass1234 | 8 | 大小写+数字 | 中等 | 🟠 橙色 | 66% |

## 安全建议解读

### 1️⃣ 至少 8 位字符，建议 12 位以上

**为什么？**
- 8 位是最低安全标准
- 12 位可抵御大多数攻击
- 每增加一位，安全性呈指数增长

**破解时间对比**:
```
8 位纯数字:    几秒钟
8 位字母+数字: 几小时
12 位混合字符: 数千年
```

### 2️⃣ 包含大小写字母、数字和特殊字符

**字符集大小**:
```
仅小写字母 (a-z):        26 种
小写+数字 (a-z, 0-9):    36 种
大小写+数字:             62 种
大小写+数字+特殊字符:     95+ 种  ◀─ 最安全
```

**示例转换**:
```
❌ password      (仅小写)
⚠️ Password      (加大写)
✅ Password123   (加数字)
✅✅ Pass@123!    (加特殊字符) ◀─ 推荐
```

### 3️⃣ 不要使用生日、电话等易猜密码

**常见弱密码列表** (请避免):
```
❌ 19900101          (生日)
❌ 13812345678       (电话)
❌ xiaoming          (姓名)
❌ 123456789         (连续数字)
❌ password123       (常见词+数字)
❌ qwerty123         (键盘顺序)
❌ iloveyou          (常见短语)
❌ admin123          (默认密码)
```

**为什么危险？**
- 黑客有庞大的弱密码字典
- 社交工程可获取个人信息
- 暴力破解会优先尝试这些

### 4️⃣ 不要与其他网站使用相同密码

**风险场景**:
```
某网站 A: 使用密码 "MyPass123"
    ↓
网站 A 被黑客攻击
    ↓
密码数据库泄露
    ↓
黑客尝试在其他网站使用相同密码
    ↓
如果您的钱包也用 "MyPass123"
    ↓
资产被盗！
```

**正确做法**:
```
网站 A:    MyPass123@SiteA
网站 B:    MyPass456@SiteB
钱包:      MyWallet#2024$Secure  ◀─ 完全独特
```

## 错误提示处理

### 常见错误信息

#### 1. "请输入密码"
```
原因: 密码输入框为空
解决: 输入至少 8 位密码
```

#### 2. "密码至少需要 8 位字符"
```
原因: 密码少于 8 位
解决: 增加字符，达到 8 位以上
```

#### 3. "两次输入的密码不一致"
```
原因: 确认密码与密码不匹配
解决: 重新输入确认密码，确保完全一致
```

### 按钮状态

#### 禁用状态 (灰色)
```
条件:
- 密码 < 8 位，或
- 确认密码不匹配

状态:
- 按钮颜色: 灰色
- 无法点击
- 无渐变效果
```

#### 启用状态 (紫色)
```
条件:
- 密码 >= 8 位
- 确认密码匹配

状态:
- 按钮颜色: 紫色渐变
- 可以点击
- 带阴影效果
```

## 操作技巧

### 查看密码

点击输入框右侧的 **眼睛图标** 👁：
- 默认: 隐藏密码 (显示 * 号)

**建议**:
- 输入时隐藏（防止偷窥）
- 确认时显示（检查正确性）

### 复制粘贴

- ✅ 可以从密码管理器粘贴
- ✅ 可以在两个输入框间复制
- ⚠️ 注意周围是否有人

### 返回修改

点击左上角 **"< 创建钱包"**:
- 返回欢迎页面
- 密码不会保存
- 可以重新开始

## 密码管理建议

### 推荐工具

**密码管理器**:
```
1Password     ⭐⭐⭐⭐⭐
LastPass      ⭐⭐⭐⭐
Bitwarden     ⭐⭐⭐⭐⭐ (开源)
KeePass       ⭐⭐⭐⭐⭐ (本地)
```

**使用方法**:
1. 在密码管理器中生成强密码
2. 保存到钱包类别
3. 添加备注（如：Memopark 钱包）
4. 设置主密码保护

### 备份策略

**方式 1: 纸质备份**
```
1. 在纸上写下密码
2. 密封在信封中
3. 存放在安全地方（保险箱）
4. 告知可信任的家人位置
```

**方式 2: 数字备份**
```
1. 使用加密的密码管理器
2. 定期导出加密备份
3. 保存在多个设备
4. 使用强主密码保护
```

**方式 3: 分散备份**
```
1. 将密码分成多段
2. 分别保存在不同位置
3. 需要全部段才能恢复
4. 降低单点泄露风险
```

## 故障排查

### 问题 1: 强度一直显示"弱"

**检查**:
- 密码长度是否 >= 8 位？
- 是否包含字母和数字？
- 是否包含大小写？

### 问题 2: 确认密码一直不匹配

**检查**:
- 是否有多余空格？
- 大小写是否完全一致？
- 是否启用了大写锁定键？

### 问题 3: 继续按钮无法点击

**检查**:
- 密码是否至少 8 位？
- 确认密码是否一致？
- 是否看到绿色"✓ 密码匹配"？

### 问题 4: 点击继续没反应

**尝试**:
1. 刷新页面 (F5)
2. 检查浏览器控制台错误
3. 确认网络连接正常
4. 清除浏览器缓存
