# 纪念馆与逝者概念说明

> **文档目的**: 明确纪念馆（Grave）与逝者（Deceased）的区别  
> **更新时间**: 2024-01-15

---

## 📖 核心概念

### 纪念馆（Grave/Memorial Hall）

**定义：**
- 纪念馆是一个**纪念空间/容器**的概念
- 类似于现实中的墓地、祠堂、纪念堂
- 可以包含一个或多个逝者的信息

**特点：**
- 📍 有固定的ID（10位数字编号）
- 🏛️ 可隶属于某个园区
- 👥 可容纳多个逝者档案
- 🔒 创建后名称不可修改
- ♾️ 永久保存在区块链上

**适用场景：**
1. **个人纪念馆** - 为单个逝者建立
2. **家族纪念馆** - 家族成员集体纪念
3. **夫妻纪念馆** - 夫妻合葬纪念
4. **团体纪念馆** - 特定团体的集体纪念

---

### 逝者（Deceased）

**定义：**
- 逝者是**具体人物**的概念
- 包含逝者的详细个人信息
- 必须关联到某个纪念馆

**特点：**
- 👤 详细的个人信息（姓名、生卒年月等）
- 🖼️ 可上传个人照片
- 📝 可记录生平事迹
- 🔗 必须关联到纪念馆
- 👨‍👩‍👧‍👦 可设置亲属关系

**包含信息：**
- 基本信息：姓名、性别、生卒日期
- 个人资料：照片、简介、生平
- 关系信息：与其他逝者的关系
- 纪念内容：供奉、留言等

## 🔄 两者关系

### 包含关系

```
纪念馆（Memorial Hall）
    ├── 逝者1（张三）
    │   ├── 基本信息
    │   ├── 照片资料
    │   └── 生平事迹
    ├── 逝者2（李四）
    │   ├── 基本信息
    │   ├── 照片资料
    │   └── 生平事迹
    └── 逝者3（王五）
        ├── 基本信息
        ├── 照片资料
        └── 生平事迹
```

### 创建顺序

**推荐流程：**

1️⃣ **先创建纪念馆**
   - 确定纪念馆类型（个人/家族/夫妻）
   - 填写纪念馆名称
   - 提交上链

2️⃣ **再创建逝者**
   - 关联到已创建的纪念馆
   - 填写逝者详细信息
   - 上传照片和资料

3️⃣ **完善关系**
   - 设置逝者间的亲属关系
   - 构建完整的关系网络

```
❌ 不推荐：
```

**家族纪念馆：**
```

- 李氏祠堂
- 陈家纪念园
- 王氏宗祠

❌ 不推荐：
- 李家（不完整）
- 我们家族（不明确）
```

**夫妻纪念馆：**
```
✅ 推荐：
- 张三李四纪念馆

- 夫妻纪念馆（无姓名）
```

### 逝者命名

**基本原则：**
```
✅ 推荐：
- 使用真实姓名
- 添加适当称谓
- 保持庄重得体

示例：
- 张三
- 李四先生
- 王五女士
- 赵六老先生

❌ 不推荐：
- 昵称（小张、老李）
- 网名（某某大侠）
- 代号（A、B、C）
```

## 💡 使用场景示例

### 场景 1：个人纪念馆

**适用情况：**
- 为单个逝者建立纪念空间
- 逝者信息完整独立

**操作步骤：**
1. 创建纪念馆：名称填写"张三纪念馆"
2. 创建逝者：在该纪念馆下创建"张三"的档案
3. 完善信息：添加照片、生平等

**结果：**
```
张三纪念馆
    └── 张三（逝者档案）
```

### 场景 2：家族纪念馆

**适用情况：**
- 家族成员集体纪念
- 多位逝者在同一纪念馆

**操作步骤：**
1. 创建纪念馆：名称填写"李氏祠堂"
2. 创建逝者1：添加"李大爷"档案
3. 创建逝者2：添加"李奶奶"档案
4. 创建逝者3：添加"李叔叔"档案
5. 设置关系：标注亲属关系

**结果：**
```
李氏祠堂
    ├── 李大爷（逝者档案）
    ├── 李奶奶（逝者档案）
    └── 李叔叔（逝者档案）
```

### 场景 3：夫妻纪念馆

**适用情况：**
- 夫妻合葬
- 相伴一生的纪念

**操作步骤：**
1. 创建纪念馆：名称填写"张三李四纪念馆"
2. 创建逝者1：添加"张三"档案
3. 创建逝者2：添加"李四"档案
4. 设置关系：标注为夫妻关系

**结果：**
```
张三李四纪念馆
    ├── 张三（逝者档案）
    └── 李四（逝者档案）
```

## 🎯 命名最佳实践

### DO ✅

1. **使用有意义的名称**
   - 体现纪念对象
   - 便于搜索识别
   - 保持庄重得体

2. **区分纪念馆类型**
   - 个人：姓名 + 纪念馆/纪念堂
   - 家族：姓氏 + 祠堂/纪念园
   - 夫妻：双方姓名 + 纪念馆

3. **考虑搜索优化**
   - 包含关键信息
   - 避免过于复杂
   - 使用常见词汇

### DON'T ❌

1. **避免无意义名称**
   - 数字编号（001、A区）
   - 随意命名（测试、暂定）
   - 过于简单（墓、碑）

2. **避免不当用词**
   - 昵称网名
   - 不雅用语
   - 商业广告

3. **避免过长名称**
   - 超过字节限制
   - 难以记忆
   - 不便搜索

## 📋 FAQ

### Q1: 为什么要分纪念馆和逝者？

**A:** 这样设计有以下好处：
- 支持一对多关系（一个纪念馆多个逝者）
- 符合现实世界的纪念方式
- 便于家族/团体纪念管理
- 更灵活的信息组织

### Q2: 必须先创建纪念馆吗？

**A:** 是的，流程建议：
1. 先创建纪念馆（获得纪念空间）
2. 再创建逝者档案（添加具体人物）
3. 完善关系和信息

### Q3: 一个纪念馆可以有多少逝者？

**A:** 根据链上设置，理论上：
- 可以添加多个逝者
- 建议不超过 10 个（性能考虑）
- 家族纪念馆可适当增加

### Q4: 纪念馆名称可以修改吗？

**A:** 不可以
- 名称上链后永久保存
- 区块链数据不可篡改
- 请在创建时仔细核对

### Q5: 逝者信息可以修改吗？

**A:** 部分信息可以
- 照片、简介等可以更新
- 基本信息（姓名、生卒）创建后建议不改
- 可以持续完善内容

## 🎓 总结

### 关键要点

1. **纪念馆是容器，逝者是内容**
2. **先建馆，后添人**
3. **名称要有意义且庄重**
4. **创建前仔细规划**

### 推荐做法

1. **规划好纪念馆类型**（个人/家族/夫妻）
2. **选择合适的名称**（简洁、有意义）
3. **按步骤创建**（先馆后人）
4. **持续完善**（逐步添加内容）
