> **调整时间**: 2024-01-15  
> **调整原因**: 优化用户体验，突出视觉重点  
> **调整状态**: ✅ 已完成

## 📋 字段顺序调整

### 调整前顺序（不合理）

```
1. 纪念馆ID（必填）
2. 逝者姓名（必填）
3. 主图上传（可选）← 位置靠后
4. 性别（可选）
5. 出生日期（必填）
6. 离世日期（必填）
7. 完整姓名 CID（可选）
8. 外部链接（可选）
```

**问题分析：**
- ❌ 主图是视觉重点，应该更显眼
- ❌ 放在姓名后面容易被忽略
- ❌ 用户可能填完基本信息才想起上传照片

---

### 调整后顺序（优化）✅

```
1. 主图上传（可选）← 移到最上方，视觉焦点
   └─ 分隔线
2. 纪念馆ID（必填）
   ──────────────────
3. 基本信息（分组）
   ├─ 逝者姓名（必填）
   └─ 性别（可选）
   ──────────────────
4. 重要日期（分组）
   ├─ 出生日期（必填）
   └─ 离世日期（必填）
   ──────────────────
5. 扩展信息（分组，可选）
   ├─ 完整姓名 CID
   └─ 外部链接
```

**改进效果：**
- ✅ 主图在最显眼位置
- ✅ 视觉引导更自然
- ✅ 符合用户认知习惯
- ✅ 提高上传率

---

## 🎯 设计理由

### 1. **视觉优先级** ⭐⭐⭐⭐⭐

**为什么主图要在最上方？**

**心理学原理：**
- 📸 照片是最直观的识别方式
- 👁️ 用户首先关注视觉元素
- 🧠 图像记忆优于文字记忆

**参考设计：**
- 各大社交平台：头像在最上方
- 身份证/护照：照片在显眼位置
- 云上思念：也是照片优先

**实际效果：**
- 用户进入页面
- 首先看到照片上传
- 自然产生"应该上传照片"的认知
- 提高照片上传率

### 2. **操作流程优化** ⭐⭐⭐⭐⭐

**调整前的问题：**
```
用户操作流程：
1. 选择纪念馆 
2. 填写姓名
3. 填写性别
4. 才看到主图上传 ← 容易遗忘
5. "啊，还要上传照片"
6. 可能懒得往回滚动
```

**调整后的优势：**
```
用户操作流程：
1. 首先看到主图上传 ✨
2. "对，应该上传照片"
3. 上传照片（一次性完成）
4. 然后填写其他信息
5. 流程更自然
6. 不容易遗忘
```

### 3. **信息分组逻辑** ⭐⭐⭐⭐⭐

**分组设计：**

**第一部分：视觉识别**
```
📸 主图上传
└─ 最直观的识别元素
```

**第二部分：关联信息**
```
🏛️ 纪念馆ID
└─ 确定所属关系
```

**第三部分：基本信息**
```
👤 逝者姓名（必填）
⚥ 性别（可选）
```

**第四部分：时间信息**
```
📅 出生日期（必填）
📅 离世日期（必填）
```

**第五部分：扩展信息**
```
📋 完整姓名 CID（可选）
🔗 外部链接（可选）
```

### 4. **参考云上思念** ⭐⭐⭐⭐⭐

**云上思念的设计：**
- 照片/头像在显眼位置
- 视觉元素优先于文字
- 符合用户习惯
