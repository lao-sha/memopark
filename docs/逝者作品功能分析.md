## 1. 功能概述

### 1.1 设计目标

逝者作品功能是 **Phase 1: AI训练数据基础** 的核心模块，旨在：

1. ✅ **记录逝者生前创作**：保存逝者生前的各类作品（文字、音频、视频、图像等）
2. ✅ **为AI训练提供数据**：为未来的AI智能体训练提供高质量数据基础
3. ✅ **隐私保护**：支持多级隐私控制，保护逝者隐私
4. ✅ **数据完整性**：通过验证机制确保作品真实性

### 1.2 核心特性

- **多类型支持**：支持 13 种作品类型（文字、音频、视频、图像等）
- **批量上传**：支持批量上传，提高效率（最多50个/次）
- **IPFS存储**：原文件存储在IPFS，链上仅存元数据
- **AI训练授权**：支持授权作品用于AI训练
- **验证机制**：支持作品真实性验证，验证后不可修改
- **隐私分级**：4级隐私控制（公开、家人、后代、私密）
- **统计功能**：自动统计作品数量和类型分布

## 2. 作品类型系统

### 2.1 作品类型枚举

系统支持 **13 种作品类型**，分为 5 大类：

#### 2.1.1 文字类（6种）

| 类型 | 元数据 | 说明 |
|------|--------|------|
| **Literature** | 体裁、字数 | 文学作品（小说、散文、诗歌、戏剧） |
| **AcademicPaper** | 研究领域、发表期刊 | 学术论文 |
| **Diary** | 无 | 日记/随笔 |
| **Letter** | 收信人 | 书信 |
| **SocialMedia** | 平台名称、帖子类型 | 社交媒体内容 |
| **Code** | 编程语言、项目描述 | 代码/技术作品 |

#### 2.1.2 音频类（3种）

| 类型 | 元数据 | 说明 |
|------|--------|------|
| **VoiceDiary** | 时长、是否有转录 | 语音日记 |
| **Music** | 音乐类型、是否有歌词 | 音乐作品 |
| **Podcast** | 主题、是否有转录 | 播客/演讲 |

#### 2.1.3 视频类（3种）

| 类型 | 元数据 | 说明 |
|------|--------|------|
| **VideoLog** | 时长、是否有字幕 | 视频日记/Vlog |
| **Lecture** | 学科/主题、是否有字幕 | 讲座/课程 |
| **LifeClip** | 场合描述 | 生活片段 |

#### 2.1.4 图像类（2种）

| 类型 | 元数据 | 说明 |
|------|--------|------|
| **Artwork** | 媒介、风格 | 艺术作品（绘画、摄影） |
| **Design** | 设计类别 | 设计作品 |

#### 2.1.5 专业技能类（1种）

| 类型 | 元数据 | 说明 |
|------|--------|------|
| **SkillDemo** | 技能名称、描述 | 专业技能展示 |

### 2.2 类型判断方法

```rust
// 判断是否为文本类型
work_type.is_text_based()  // Literature, AcademicPaper, Diary, Letter, SocialMedia, Code

// 判断是否为音频类型
work_type.is_audio_based()  // VoiceDiary, Music, Podcast

// 判断是否为视频类型
work_type.is_video_based()  // VideoLog, Lecture, LifeClip

// 判断是否有转录文本
work_type.has_transcript()  // 音频/视频是否有转录或字幕
```

## 3. 数据结构设计

### 3.1 作品记录结构（DeceasedWork）

```rust
pub struct DeceasedWork<AccountId, BlockNumber> {
    // === 基础信息 ===
    pub work_id: u64,                    // 作品唯一ID
    pub deceased_id: u64,                // 所属逝者ID
    pub work_type: WorkType,             // 作品类型（含元数据）
    pub title: BoundedVec<u8, 200>,      // 作品标题（最多200字符）
    pub description: BoundedVec<u8, 1000>, // 作品描述（最多1000字符）
    
    // === 存储信息 ===
    pub ipfs_cid: BoundedVec<u8, 64>,    // IPFS存储地址（CID）
    pub file_size: u64,                  // 文件大小（字节）
    
    // === 时间信息 ===
    pub created_at: Option<u64>,         // 创作时间（Unix时间戳，可选）
    pub uploaded_at: BlockNumber,        // 上传时间（区块号）
    pub uploader: AccountId,              // 上传者账户
    
    // === AI相关标签 ===
    pub tags: BoundedVec<BoundedVec<u8, 50>, 20>,  // 主题标签（最多20个）
    pub sentiment: Option<i8>,           // 情感倾向（-100到100）
    pub style_tags: BoundedVec<BoundedVec<u8, 50>, 10>,  // 语言风格标签
    pub expertise_fields: BoundedVec<BoundedVec<u8, 50>, 10>,  // 专业领域标签
    
    // === 权限控制 ===
    pub privacy_level: PrivacyLevel,      // 隐私级别
    pub ai_training_enabled: bool,        // 是否授权AI训练
    pub public_display: bool,            // 是否可公开展示
    
    // === 验证信息 ===
    pub verified: bool,                   // 是否已验证
    pub verifier: Option<AccountId>,      // 验证者账户（可选）
}
```

### 3.2 隐私级别（PrivacyLevel）

| 级别 | 代码 | 说明 | 访问权限 |
|------|------|------|---------|
| **Public** | 0 | 完全公开 | 任何人可查看 |
| **Family** | 1 | 仅家人可见 | 需验证家庭关系 |
| **Descendants** | 2 | 仅后代可见 | 需验证血缘关系 |
| **Private** | 3 | 私密内容 | 仅用于AI训练，不公开展示 |

**默认级别**：`Family`（仅家人可见）

## 4. 存储架构

### 4.1 存储项设计

#### 4.1.1 主存储

| 存储项 | 类型 | 键 | 值 | 说明 |
|--------|------|-----|-----|------|
| `NextWorkId` | `StorageValue` | - | `u64` | 下一个作品ID（全局递增） |
| `DeceasedWorks` | `StorageMap` | `work_id` | `DeceasedWork` | 作品完整记录 |

#### 4.1.2 索引存储

| 存储项 | 类型 | 键 | 值 | 说明 |
|--------|------|-----|-----|------|
| `WorksByDeceased` | `StorageMap` | `deceased_id` | `BoundedVec<u64, 10000>` | 逝者作品列表（最多10000个） |
| `WorksByType` | `StorageDoubleMap` | `(deceased_id, work_type_str)` | `BoundedVec<u64, 1000>` | 按类型索引（每种类型最多1000个） |
| `AITrainingWorks` | `StorageMap` | `deceased_id` | `BoundedVec<u64, 5000>` | AI训练授权作品列表（最多5000个） |
| `WorkStatsByDeceased` | `StorageMap` | `deceased_id` | `WorkStats` | 作品统计信息 |

### 4.2 索引设计理念

**多维度索引**：
- ✅ **按逝者索引**：快速查询某个逝者的所有作品
- ✅ **按类型索引**：按作品类型筛选（用于AI训练）
- ✅ **AI训练索引**：快速获取授权AI训练的作品
- ✅ **统计信息**：实时统计作品数量和类型分布

**性能优化**：
- 使用 `BoundedVec` 限制索引大小，防止DoS攻击
- 每个逝者最多 10000 个作品
- 每种类型最多 1000 个作品
- AI训练授权最多 5000 个作品

**接口位置**: `pallets/deceased/src/lib.rs:4397`

**功能**：
- 上传单个作品到指定逝者
- 自动生成作品ID
- 更新所有相关索引
- 更新统计信息

**参数**：
```rust
pub fn upload_work(
```

**功能**：
- 批量上传多个作品（最多50个）
- 减少交易次数和Gas费用

**参数**：

**事件**：`WorksBatchUploaded { deceased_id, count, uploader }`

### 5.3 更新作品元数据（update_work）

**接口位置**: `pallets/deceased/src/lib.rs:4540`

**功能**：
- 更新作品的标题、描述、标签等元数据
- **已验证的作品无法修改**（保护数据完整性）
- IPFS CID 和文件大小不可修改

**可更新字段**：
- ✅ 标题（title）
- ✅ 描述（description）
- ✅ 主题标签（tags）
- ✅ 隐私级别（privacy_level）
- ✅ AI训练授权（ai_training_enabled）

**不可更新字段**：
- ❌ IPFS CID（确保数据完整性）
- ❌ 文件大小
- ❌ 作品类型
- ❌ 创作时间

**参数**：
```rust
pub fn update_work(
    origin: OriginFor<T>,
    work_id: u64,
    title: Option<Vec<u8>>,
    description: Option<Vec<u8>>,
    tags: Option<Vec<Vec<u8>>>,
    privacy_level_code: Option<u8>,
    ai_training_enabled: Option<bool>,
) -> DispatchResult
```

**权限要求**：
- 调用者必须是作品所属逝者的 `owner`
- 作品必须未验证（`verified = false`）

**权重**：`30_000`

**事件**：`WorkUpdated { work_id, deceased_id }`

### 5.4 删除作品（delete_work）

**接口位置**: `pallets/deceased/src/lib.rs:4602`

**功能**：
- 从链上存储中移除作品记录
- 更新所有相关索引
- 更新统计信息
- **不删除IPFS文件**（需要手动unpinning）

**参数**：
```rust
pub fn delete_work(
    origin: OriginFor<T>,
    work_id: u64,
) -> DispatchResult
```

**权限要求**：
- 调用者必须是作品所属逝者的 `owner`

**注意事项**：
- IPFS文件不会被自动删除
- 删除后作品ID无法恢复使用

**权重**：`40_000`

**事件**：`WorkDeleted { work_id, deceased_id }`

### 5.5 验证作品（verify_work）

**接口位置**: `pallets/deceased/src/lib.rs:4646`

**功能**：
- 标记作品为"已验证"状态
- 验证后的作品无法修改（保护数据完整性）
- 记录验证者信息

**权限要求**：
- 逝者的 `owner` 可以验证
- 委员会成员可以验证（通过 `GovernanceOrigin`）

**参数**：
```rust
pub fn verify_work(
    origin: OriginFor<T>,
    work_id: u64,
) -> DispatchResult
```

**验证效果**：
- 设置 `verified = true`
- 记录 `verifier` 账户
- 后续调用 `update_work` 会返回 `WorkAlreadyVerified` 错误

**权重**：`20_000`

**事件**：`WorkVerified { work_id, deceased_id, verifier }`

## 6. AI训练支持

### 6.1 AI训练授权机制

**授权字段**：
- `ai_training_enabled: bool` - 是否授权用于AI训练

**授权效果**：
- 授权后，作品会被添加到 `AITrainingWorks` 索引
- AI训练服务可以快速查询授权作品列表
- 未授权作品不会出现在AI训练索引中

### 6.2 AI训练价值评估

**评估标准**：
```rust
// 检查作品是否对AI训练有价值
pub fn is_ai_training_valuable(&self) -> bool {
    self.ai_training_enabled
        && (self.work_type.is_text_based() || self.work_type.has_transcript())
}
```

**权重计算**：
```rust
// 获取作品的AI训练权重
pub fn ai_training_weight(&self) -> u8 {
    if !self.ai_training_enabled {
        return 0;
    }
    
    if self.work_type.is_text_based() {
        100  // 文本类型：最高价值
    } else if self.work_type.has_transcript() {
        80   // 有转录的音频/视频：高价值
    } else {
        20   // 其他类型：辅助价值
    }
}
```

**权重规则**：
- **文本类型**：100（最高价值）
- **有转录的音频/视频**：80（高价值）
- **其他类型**：20（辅助价值）
- **未授权**：0（无价值）

### 6.3 AI训练数据导出

**索引支持**：
- `AITrainingWorks` - 快速查询授权作品列表
- `WorksByType` - 按类型筛选（优先文本类型）
- `WorkStats` - 评估训练数据量

**导出流程**：
1. 查询 `AITrainingWorks` 获取授权作品ID列表
2. 遍历ID列表，从 `DeceasedWorks` 获取完整作品信息
3. 根据 `ai_training_weight()` 排序，优先使用高权重作品
4. 从IPFS获取原文件内容
5. 生成训练数据集

## 7. 隐私控制机制

### 7.1 隐私级别定义

| 级别 | 代码 | 访问权限 | 使用场景 |
|------|------|---------|---------|
| **Public** | 0 | 任何人可查看 | 公开作品、纪念文章 |
| **Family** | 1 | 仅家人可见 | 家庭内部内容（默认） |
| **Descendants** | 2 | 仅后代可见 | 留给后代的遗产 |
| **Private** | 3 | 仅AI训练 | 私密内容，不公开展示 |

### 7.2 权限检查逻辑

**访问控制**：
- 墓地所有者（`owner`）始终可以访问所有级别
- 其他用户需要验证关系（家庭关系、血缘关系）
- AI训练服务需要专门授权（`ai_training_enabled = true`）

**实现位置**：
- 权限检查在查询接口中实现（前端或中间件）
- 链上存储隐私级别，链下验证访问权限

### 7.3 隐私级别转换

**转换规则**：
- 未验证的作品可以修改隐私级别
- 已验证的作品无法修改隐私级别（保护数据完整性）
- 降低隐私级别（Public → Family）需要谨慎

## 8. 权限管理

### 8.1 操作权限

| 操作 | 权限要求 | 说明 |
|------|---------|------|
| **上传作品** | 逝者 `owner` | 仅墓地所有者可以上传 |
| **更新作品** | 逝者 `owner` + 未验证 | 已验证作品无法修改 |
| **删除作品** | 逝者 `owner` | 仅墓地所有者可以删除 |
| **验证作品** | 逝者 `owner` 或 治理权限 | 委员会可以验证 |

### 8.2 权限检查实现

**检查流程**：
```rust
// 1. 获取作品记录
let work = DeceasedWorks::<T>::get(work_id)
    .ok_or(Error::<T>::WorkNotFound)?;

// 2. 获取逝者记录
let deceased = DeceasedOf::<T>::get(work.deceased_id)
    .ok_or(Error::<T>::DeceasedNotFound)?;

// 3. 检查拥有者权限
ensure!(deceased.owner == who, Error::<T>::WorkNotAuthorized);

// 4. 检查验证状态（更新时）
ensure!(!work.verified, Error::<T>::WorkAlreadyVerified);
```

**位置**: `pallets/deceased/src/lib.rs:2173`

**功能**：
- 标记作品为已验证
- 记录验证者信息

**关键步骤**：
1. 获取作品记录
2. 检查是否已验证
3. 设置 `verified = true`
4. 记录 `verifier` 账户

### 10.2 错误处理

| 错误 | 触发条件 | 说明 |
|------|---------|------|
| `WorkNotFound` | 作品ID不存在 | 查询、更新、删除时 |
| `TooManyWorks` | 作品数量超过限制 | 单个逝者超过10000个 |
| `TitleTooLong` | 标题超过200字符 | 上传/更新时 |
| `DescriptionTooLong` | 描述超过1000字符 | 上传/更新时 |
| `TagsTooMany` | 标签超过20个或单个超过50字符 | 上传/更新时 |
| `WorkNotAuthorized` | 无权限操作该作品 | 非owner尝试修改/删除 |
| `WorkAlreadyVerified` | 作品已验证，无法修改 | 尝试修改已验证作品 |

### 10.3 事件设计

| 事件 | 触发时机 | 包含信息 |
|------|----------|----------|
| `WorkUploaded` | 作品上传成功 | `work_id, deceased_id, work_type_str, uploader` |
| `WorksBatchUploaded` | 批量上传成功 | `deceased_id, count, uploader` |
| `WorkUpdated` | 作品元数据更新 | `work_id, deceased_id` |
| `WorkDeleted` | 作品删除 | `work_id, deceased_id` |
| `WorkVerified` | 作品验证 | `work_id, deceased_id, verifier` |
