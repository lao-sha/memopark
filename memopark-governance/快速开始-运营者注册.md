# 运营者注册功能 - 快速开始

## 🚀 访问入口

1. 启动治理页面：
   ```bash
   cd /home/xiaodong/文档/memopark/memopark-governance
   npm run dev
   ```

2. 浏览器访问：`http://localhost:5173/operator-management`

3. 侧边栏点击：**运营者管理** 菜单项

---

## 📋 注册步骤（5 步完成）

### 第 1 步：准备 IPFS 环境

```bash
# 启动 IPFS 节点
ipfs daemon

# 获取 Peer ID
ipfs id
# 输出示例：QmXXXXXX...（复制此 ID）
```

### 第 2 步：准备 IPFS Cluster

```bash
# 启动 Cluster
ipfs-cluster-service daemon

# 确认 API 端点
curl http://your-server:9094/version
```

### 第 3 步：填写注册表单

| 字段 | 填写内容 |
|------|----------|
| IPFS Peer ID | 步骤 1 获取的 Peer ID |
| 存储容量 | 例如：`1000`（GB） |
| 集群端点 | 例如：`http://your-server:9094` |
| 保证金 | 例如：`1000`（MEMO） |

### 第 4 步：提交交易

1. 点击"提交注册"
2. 钱包弹出签名确认
3. 确认并等待上链（约 6-12 秒）

### 第 5 步：验证注册成功

- ✅ 收到"注册成功"提示
- ✅ 列表中出现您的账户
- ✅ 状态显示为"活跃"

## 🔧 治理功能（需要权限）

### 执行奖励分配

**条件**：Root 账户或技术委员会成员

**步骤**：
1. 点击"执行奖励分配"
2. 输入分配金额（0 = 全部余额）
3. 确认并签名
4. 自动按权重分配给所有活跃运营者

**权重公式**：
```
权重 = 已存储字节数 × (探测成功次数 / 总探测次数)
```

## ⚠️ 注意事项

1. **保证金要求**：至少 1000 MEMO
2. **容量要求**：至少 100 GB
3. **在线率要求**：建议 > 95%
4. **网络要求**：端点需可从链端访问

## 🐛 常见问题

### Q1: 注册后没有收到任务？

**检查**：
- [ ] IPFS Cluster 是否运行正常
- [ ] 端点是否可访问
- [ ] 防火墙是否开放端口
- [ ] 状态是否为"活跃"

### Q2: 可靠性一直很低？

**改进**：
- 使用稳定的云服务器
- 增加节点资源（CPU/内存）
- 优化网络连接
- 检查防火墙规则
